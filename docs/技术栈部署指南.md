# OA管理系统技术栈部署指南

## 📋 部署前准备

### 必需软件清单

| 软件 | 版本要求 | 用途 | 下载地址 |
|------|---------|------|---------|
| JDK | 17+ | Java运行环境 | https://www.oracle.com/java/technologies/downloads/ |
| MySQL | 8.0+ | 数据库 | https://dev.mysql.com/downloads/mysql/ |
| Nacos | 2.0+ | 配置/注册中心 | https://nacos.io/zh-cn/ |
| Redis | 6.0+ | 缓存中间件 | https://redis.io/download |
| Elasticsearch | 7.15+ | 全文检索引擎 | https://www.elastic.co/downloads/elasticsearch |
| Node.js | 16+ | 前端运行环境 | https://nodejs.org/ |

**Elasticsearch IK分词器**（必须安装，用于中文检索）
```bash

# 重启Elasticsearch
```

## 🚀 部署步骤

### 第一步：数据库配置

#### 1. 创建数据库
```sql
CREATE DATABASE oa_system DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;
```

#### 2. 导入数据库脚本（按顺序执行）
```bash
# 基础表结构
mysql -u root -p oa_system < database/oa_system.sql

# 角色权限升级
mysql -u root -p oa_system < database/upgrade_v1.1_role_permission.sql

# 技术栈集成
mysql -u root -p oa_system < database/upgrade_v2.0_nacos_redis_es.sql
```

#### 3. 修改数据库密码（如需要）
```sql
-- 查看当前用户密码加密格式
SELECT id, username, password FROM t_employee WHERE id = 1;

-- 更新密码（示例：将admin的密码改为加密后的123456）
-- 注意：需要使用PasswordEncoder生成加密密码
```

---

### 第二步：Nacos配置

#### 1. 启动Nacos（单机模式）
```bash
# Windows
cd nacos/bin
startup.cmd -m standalone

# Linux/Mac
cd nacos/bin
sh startup.sh -m standalone
```

#### 2. 访问Nacos控制台
- 地址: http://localhost:8848/nacos
- 用户名: nacos
- 密码: nacos

#### 3. 创建配置文件

在Nacos控制台创建以下配置：

**配置1：oa-management-service.yml**
- Data ID: `oa-management-service.yml`
- Group: `DEFAULT_GROUP`
- 配置格式: `YAML`
- 配置内容:
```yaml
spring:
  datasource:
    driver-class-name: com.mysql.cj.jdbc.Driver
    url: jdbc:mysql://localhost:3306/oa_system?useUnicode=true&characterEncoding=utf8&useSSL=false&serverTimezone=Asia/Shanghai
    username: root
    password: 'zyx251026'

  redis:
    host: localhost
    port: 6379
    password:
    database: 0
    timeout: 3000ms
    lettuce:
      pool:
        max-active: 8
        max-wait: -1ms
        max-idle: 8
        min-idle: 0

  elasticsearch:
    uris: http://localhost:9200
    username:
    password:
    connection-timeout: 5s
    socket-timeout: 30s

  cache:
    type: redis
    redis:
      time-to-live: 3600000
      cache-null-values: false
      use-key-prefix: true
      key-prefix: oa_cache:
```

**配置2：common-config.yml**（公共配置）
- Data ID: `common-config.yml`
- Group: `DEFAULT_GROUP`
- 配置格式: `YAML`
- 配置内容:
```yaml
# 公共配置
jwt:
  secret: oa-management-secret-key-2026
  expiration: 86400000

# 日志配置
logging:
  level:
    com.southwind: debug
```

### 第三步：Redis配置

#### 1. 启动Redis
```bash
# Windows
redis-server.exe redis.windows.conf

# Linux/Mac
redis-server
```

#### 2. 验证Redis启动
```bash
redis-cli
127.0.0.1:6379> ping
PONG
```

#### 3. （可选）设置Redis密码
```bash
# 编辑redis.conf
requirepass your_password

# 连接Redis
redis-cli -a your_password
```

### 第五步：后端启动

#### 1. 配置本地配置文件

编辑 `backend/oa-springboot/src/main/resources/application.yml`，确保以下配置正确：
```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/oa_system
    username: root
    password: 'zyx251026'  # 修改为你的MySQL密码

  redis:
    host: localhost
    port: 6379
    password:  # 如有密码请填写

  elasticsearch:
    uris: http://localhost:9200
    username:  # 如有用户名请填写
    password:  # 如有密码请填写
```

#### 2. 编译项目
```bash
cd backend/oa-springboot
mvn clean install
```

#### 3. 启动后端
```bash
mvn spring-boot:run
```

#### 4. 验证启动成功

查看控制台日志，确认以下内容：
- ✅ Nacos注册成功：`register service to nacos`
- ✅ Redis连接成功：`Redis initialized`
- ✅ ES索引创建成功：`同步员工数据到ES成功`
- ✅ 应用启动成功：`Started OaManagementApplication`

## 🔧 常见问题排查

### 问题1：Nacos服务未注册

**症状**: Nacos控制台看不到服务

**排查**:
1. 检查Nacos是否启动：访问 http://localhost:8848/nacos
2. 检查bootstrap.yml配置：确保server-addr正确
3. 查看后端日志：搜索"nacos"关键字

### 问题2：Redis连接失败

**症状**: 控制台报错"Unable to connect to Redis"

**排查**:
1. 检查Redis是否启动：`redis-cli ping`
2. 检查application.yml配置：确保host和port正确
3. 如有密码，检查password配置

### 问题3：Elasticsearch索引未创建

**症状**: ES中看不到索引

**排查**:
1. 检查ES是否启动：`curl http://localhost:9200`
2. 检查IK分词器是否安装：`curl http://localhost:9200/_cat/plugins`
3. 查看后端日志：搜索"同步员工数据到ES"关键字

### 问题4：前端无法访问后端

**症状**: 前端报错"Network Error"

**排查**:
1. 检查后端是否启动：访问 http://localhost:8080
2. 检查跨域配置：CrosConfiguration是否生效
3. 检查axios配置：baseURL是否正确

## 📊 性能优化建议

### 1. Redis缓存优化
- 设置合理的过期时间（默认1小时）
- 热点数据使用`@Cacheable`注解
- 定期清理缓存记录表

### 2. Elasticsearch优化
- 定期重建索引（使用定时任务）
- 调整分片数量和副本数量
- 监控索引大小和搜索性能

### 3. MySQL优化
- 定期分析慢查询日志
- 优化索引结构
- 考虑读写分离
