## 1️⃣ Nacos 集成指南

### 功能说明
- ✅ 服务注册与发现
- ✅ 配置中心（动态配置）
- ✅ 配置热更新

### 环境准备

#### 安装 Nacos Server
```bash
# 下载 Nacos
https://github.com/alibaba/nacos/releases

# Windows 启动
cd nacos/bin
startup.cmd -m standalone

# 访问控制台
http://localhost:8848/nacos
用户名/密码: nacos/nacos
```

#### 添加依赖
```xml
<!-- pom.xml -->
<dependency>
    <groupId>com.alibaba.cloud</groupId>
    <artifactId>spring-cloud-starter-alibaba-nacos-discovery</artifactId>
    <version>2021.0.5.0</version>
</dependency>
<dependency>
    <groupId>com.alibaba.cloud</groupId>
    <artifactId>spring-cloud-starter-alibaba-nacos-config</artifactId>
    <version>2021.0.5.0</version>
</dependency>
```

#### 配置文件
```yaml
# application.yml
spring:
  cloud:
    nacos:
      discovery:
        server-addr: localhost:8848
        namespace: public
      config:
        server-addr: localhost:8848
        namespace: public
        file-extension: yaml
        shared-configs:
          - data-id: common.yaml
            refresh: true
```

#### Nacos 控制台配置
```yaml
# Data ID: oa-management-service.yaml
# Group: DEFAULT_GROUP

spring:
  datasource:
    url: jdbc:mysql://localhost:3306/oa_system
    username: root
    password: zyx251026

  redis:
    host: localhost
    port: 6379

# 其他配置...
```

#### 验证集成

## 2️⃣ Redis 集成指南

### 功能说明
- ✅ 登录用户信息缓存
- ✅ 热门数据缓存
- ✅ 分页数据缓存
- ✅ 接口限流
- ✅ 验证码缓存

### 环境准备

#### 安装 Redis
```bash
# Windows 安装
https://github.com/microsoftarchive/redis/releases

# 启动 Redis
redis-server.exe

# 测试连接
redis-cli.exe
ping
```

#### 依赖（已添加）
```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-data-redis</artifactId>
</dependency>
<dependency>
    <groupId>org.apache.commons</groupId>
    <artifactId>commons-pool2</artifactId>
</dependency>
```

#### 配置文件
```yaml
# application.yml
spring:
  redis:
    host: localhost
    port: 6379
    password:          # 如果有密码
    database: 0
    timeout: 3000ms
    lettuce:
      pool:
        max-active: 8
        max-wait: -1ms
        max-idle: 8
        min-idle: 0

  cache:
    type: redis
    redis:
      time-to-live: 3600000  # 1小时
      cache-null-values: false
      use-key-prefix: true
      key-prefix: oa_cache:
```

#### 使用示例
```java
// RedisCacheUtil.java 已实现

// 1. 基础缓存操作
@Autowired
private RedisCacheUtil redisCacheUtil;

// 设置缓存
redisCacheUtil.set("user:1", user, 3600);

// 获取缓存
User user = redisCacheUtil.get("user:1");

// 删除缓存
redisCacheUtil.delete("user:1");

// 批量删除
redisCacheUtil.deleteByPattern("employee:*");

// 2. Spring Cache 注解
@Cacheable(value = "employee", key = "#id")
public Employee getById(Integer id) {
    return employeeMapper.selectById(id);
}

@CacheEvict(value = "employee", key = "#id")
public boolean update(Employee employee) {
    return employeeMapper.updateById(employee) > 0;
}
```

#### 验证集成
```java
@RestController
public class TestController {
    @Autowired
    private RedisCacheUtil redisCacheUtil;
    
    @GetMapping("/test/redis")
    public String testRedis() {
        redisCacheUtil.set("test:key", "Hello Redis", 60);
        return redisCacheUtil.get("test:key");
    }
}
```

## 3️⃣ Elasticsearch 集成指南

### 功能说明
- ✅ 员工全文检索
- ✅ 公告全文检索
- ✅ 模糊查询
- ✅ 高亮显示
- ✅ 数据同步

### 环境准备

#### 安装 Elasticsearch
```bash
# 下载 ES
https://www.elastic.co/downloads/elasticsearch

# Windows 启动
cd elasticsearch/bin
elasticsearch.bat

# 访问测试
http://localhost:9200

# 安装 IK 中文分词器
./bin/elasticsearch-plugin install https://github.com/medcl/elasticsearch-analysis-ik/releases/download/v7.17.0/elasticsearch-analysis-ik-7.17.0.zip
```

#### 依赖（已添加）
```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-data-elasticsearch</artifactId>
</dependency>
```

#### 配置文件
```yaml
# application.yml
spring:
  elasticsearch:
    uris: http://localhost:9200
    username:           # 如果有
    password:           # 如果有
    connection-timeout: 5s
    socket-timeout: 30s
```

#### 创建索引
```json
# 员工索引
PUT /employee_index
{
  "mappings": {
    "properties": {
      "id": {"type": "integer"},
      "username": {"type": "text", "analyzer": "ik_max_word"},
      "name": {"type": "text", "analyzer": "ik_max_word"},
      "departmentName": {"type": "text", "analyzer": "ik_max_word"},
      "position": {"type": "text", "analyzer": "ik_max_word"}
    }
  }
}

# 公告索引
PUT /announcement_index
{
  "mappings": {
    "properties": {
      "id": {"type": "integer"},
      "title": {"type": "text", "analyzer": "ik_max_word"},
      "content": {"type": "text", "analyzer": "ik_max_word"},
      "publisherName": {"type": "text", "analyzer": "ik_max_word"}
    }
  }
}
```

#### 使用示例
```java
// EmployeeSearchService.java 已实现

// 1. 全文检索

// 2. 部门搜索

// 3. 数据同步
employeeSearchService.syncEmployeeToEs(employeeDocument);

// 4. SearchController 已实现搜索接口
// GET /search/employee?keyword=张三

```

## 🐛 常见问题

### Q1: Nacos 连接失败
```
解决方案：
1. 检查 Nacos Server 是否启动
2. 检查端口8848是否被占用
3. 检查防火墙设置
```

### Q2: Redis 连接超时
```
解决方案：
1. 检查 Redis Server 是否启动
2. 检查配置的 host 和 port
3. 如有密码，确认 password 配置
```

### Q3: Elasticsearch 索引不存在
```
解决方案：
1. 先创建索引（见上文）
2. 调用同步接口：POST /api/database/es/sync
3. 检查 ES Server 是否启动
```
